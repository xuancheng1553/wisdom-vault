# 一页纸总结：系统思维

## 一句话定义
> 整体不等于部分之和。要理解一个系统，不能只看部件，要看部件之间的相互关系。

## 核心概念
1. **全局视角**：看一个公司不能只看财务数据，要看人、文化、竞争、行业的关系
2. **相互依赖**：系统中的一个变化会影响其他部分，有些影响是你想不到的
3. **反馈回路**：系统中的因果关系往往不是线性的，有正反馈和负反馈
4. **延迟效应**：系统的变化往往有延迟，今天做的事可能半年后才看到效果
5. **芒格的格栅理论**：系统思维的极致——用多个学科的模型看同一个系统

## 真实案例
- 你的社群：内容、用户、价格、口碑、竞争——这是一个系统，改变一个会影响所有
- 你的资产配置：各类资产之间相互关联，一个变了其他的也会变
- 芒格的可口可乐分析：从品牌、渠道、配方、规模效应、文化多角度同时分析

## 常见错误
| 错误 | 表现 |
|------|------|
| 只看局部 | 只看一个指标做判断 |
| 忽视延迟 | 今天做了调整明天就想看到结果 |
| 简单归因 | 把结果归因于一个原因 |
| 忽视外部影响 | 以为系统是封闭的 |

## 检查清单
- [ ] 我看到了系统的全貌还是只看到了部分？
- [ ] 我的改变会怎么影响系统的其他部分？
- [ ] 我考虑了延迟效应吗？
- [ ] 我用了多少种思维模型来看这个问题？
