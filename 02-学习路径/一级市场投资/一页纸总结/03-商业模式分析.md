# 一页纸总结：商业模式分析

## 一句话定义
> 商业模式是回答"钱是怎么赚的"——谁付钱、为什么付、付多少、成本多少、能不能规模化。

## 核心概念
1. **价值主张**：你为客户解决了什么根本问题？客户为什么选你不选别人？
2. **收入模型**：卖什么、怎么收费——SaaS订阅→高频复购、交易抽佣→规模效应、硬件销售→一次性
3. **成本结构**：固定成本vs可变成本、研发占比、营销费用、基础设施成本
4. **单位经济模型**：LTV（客户终身价值）和CAC（客户获取成本）——LTV/CAC > 3 是一个健康的项目
5. **可规模化**：每多一个客户，边际成本是否趋近于零？
6. **护城河**：技术壁垒、网络效应、规模优势、品牌、转换成本、先发优势

## 真实案例
- **SaaS模式**（如飞书/钉钉）：收入=订阅费×客户数×续费率。CAC较高（销售团队），但LTV很长（续费率80%+），LTV/CAC通常>5，是好模式
- **共享单车**：收入=骑行费（低频低价）、CAC=单车成本（几百元）+运维成本（月月有）、LTV被高估（用户复骑率低）。LTV/CAC < 1，是坏模式

## 常见错误
| 错误 | 表现 | 后果 |
|------|------|----------|
| 收入模型证伪 | 没想清楚谁付钱 | 创业公司没收入 |
| CAC被低估 | 只算了广告费，没算销售团队成本 | 单位经济模型失真 |
| LTV被高估 | 假设客户留存率远超实际 | 估值泡沫 |
| 伪护城河 | 以为先发优势就是护城河 | BAT一来就死 |

## 检查清单
- [ ] 客户愿意付钱的核心原因是什么？
- [ ] 收入模型是高频低频？复购率多少？
- [ ] LTV/CAC > 3 吗？
- [ ] 边际成本趋近于零吗？
- [ ] 护城河到底是什么？（技术、网络效应、品牌、规模？）

## 5个自测题
Q1：单位经济模型中最重要的两个指标是什么？
A：LTV（客户终身价值）和CAC（客户获取成本），LTV/CAC > 3才是健康的。

Q2：什么商业模式是最糟糕的？
A：LTV < CAC、收入低频、边际成本不下降、没有护城河——比如共享单车。

Q3：为什么"先发优势"不是真正的护城河？
A：如果没有技术或网络效应保护，后发者可以复制你。真正的护城河是"后发者不可能轻易复制的东西"。

Q4：分析一个你知道的公司模式，它的LTV/CAC大概是多少？
A：（开放题）练习测算。

Q5：如果这个项目今天开始做，最大的不确定性是什么？
A：（开放题）找到最关键的假设。
